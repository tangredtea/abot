# EXPERIMENTS.md

This file tracks your self-improvement experiments and configuration changes.

## Format

```
### [DATE] Brief title
**Trigger**: What prompted this?
**Observation**: What did you notice?
**Action**: What change did you make?
**Expected outcome**: How will this help?
```

## Example

```
### 2026-03-06 Improved error handling in file operations
**Trigger**: User reported confusing error messages when files don't exist
**Observation**: Error messages were too technical, didn't suggest solutions
**Action**: Updated error messages to be more user-friendly and actionable
**Expected outcome**: Users will understand what went wrong and how to fix it
```
