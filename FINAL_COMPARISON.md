# abot vs Memoh-v2 深度对比分析（最终版）

> 基于最新代码和模板内容的完整对比
> 生成时间：2026-03-06

## 📊 第一部分：架构对比

### Memoh-v2 架构
```
三服务分离 + 容器隔离：
┌─────────────────────────────────────┐
│ Go Server (8080)                    │
│ - REST API, Auth, DB                │
│ - 容器管理 (containerd)              │
│ - 记忆系统 (Qdrant + BM25)          │
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│ Agent Gateway (8081) - Bun/TS      │
│ - LLM 推理, 工具执行                 │
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│ Web Frontend (8082) - Vue 3        │
└─────────────────────────────────────┘

每个 Bot 独立容器：
- /data/ - 私有文件 (IDENTITY/SOUL/TOOLS)
- /shared/ - 跨 Bot 共享
- containerd 隔离
```

### abot 架构
```
模块化单体 + 中心化存储：
┌─────────────────────────────────────┐
│ abot (纯 Go)                        │
│ - API Server                        │
│ - Agent Engine                      │
│ - Provider Abstraction              │
│ - Tool System                       │
│ - Channel Adapters                  │
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│ MySQL 中心化存储                     │
│ - workspace_docs (SOUL/IDENTITY)    │
│ - user_workspace_docs (USER)        │
│ - 容器无状态                         │
└─────────────────────────────────────┘

部署模式：
- CLI: 单进程
- API: + MySQL
- Web: + Next.js
```

**关键差异**：
- Memoh-v2: 文件系统存储 + 容器有状态
- abot: MySQL 存储 + 容器无状态

## 📊 第五部分：架构优缺点对比

### Memoh-v2 优势

**1. 容器隔离**
- ✅ 每个 Bot 独立环境
- ✅ 完全隔离，安全性高
- ✅ 支持快照回滚
- ❌ 资源占用大
- ❌ 启动慢

**2. 文件系统存储**
- ✅ 直观，易于理解
- ✅ 支持 /shared/ 跨 Bot 协作
- ✅ 容器有状态，持久化简单
- ❌ 难以扩展
- ❌ 备份复杂

**3. 三服务架构**
- ✅ 职责分离
- ✅ 可独立扩展
- ❌ 部署复杂
- ❌ 跨语言维护

### abot 优势

**1. 中心化存储**
- ✅ 容器无状态，易于扩展
- ✅ 备份简单（MySQL）
- ✅ 多租户隔离完善
- ✅ 易于迁移和复制
- ❌ 无跨 Agent 文件协作

**2. 纯 Go 实现**
- ✅ 维护成本低
- ✅ 性能好
- ✅ 编译型，部署简单
- ✅ 跨平台兼容

**3. 灵活部署**
- ✅ CLI/API/Web 多模式
- ✅ 资源占用小
- ✅ 启动快
- ✅ 适合各种场景
