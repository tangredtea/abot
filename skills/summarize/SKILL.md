# Summarize

Condense long content into structured summaries.

## When to Use

- User provides a long document and asks for a summary
- Conversation history needs condensing
- Article or web page content needs key points extracted

## Output Format

Adapt format to content type:

- **Articles/docs**: Key points as bullet list + one-paragraph summary
- **Conversations**: Decisions made, action items, open questions
- **Technical content**: Problem, approach, solution, caveats

## Guidelines

- Preserve factual accuracy — never invent details
- Keep summaries under 20% of original length
- Highlight actionable items separately
- For very long content (>10k words), offer a tiered summary: executive summary first, then section-by-section on request
